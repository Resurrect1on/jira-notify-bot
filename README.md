# Сервис для интеграции Jira с целью отправки уведомлений о новых действиях в проекте

## Описание

Сервис отвечает за отправку уведомлений по проектам *SUPP*, *SUZI*, *EX*, *SZO* о новых действиях в обращениях.

[Redoc](http://example.com/api/docs/redoc)

[Swagger](http://example.com/api/docs/swagger)

## Запуск сервиса

Для успешного запуска потребуются загруженные в среду переменные окружения или файл `.env` с ними.

```json
{
  "VERSION": "Версия приложения",
  "MATTERMOST_URL": "Ссылка до API мессенджера Mattermost",
  "SUZI_PROJECT_HOOK": "Интеграционный хук до чата проекта SUZI",
  "MONITORING_HOOK": "Интеграционный хук до чата проекта Monitoring",
  "SUPPORT_PROJECT_HOOK": "Интеграционный хук до чата проекта Support",
  "EX_PROJECT_HOOK": "Интеграционный хук до чата проекта Exploitation",
  "WB_PROJECT_HOOK": "Интеграционный хук до чата проекта WB (SZO)",
  "CLIENT_VOICE_HOOK": "Интеграционный хук до чата проекта CLIENT-VOICE (SZO)",
}
```

Более подробные данные о настройках переменных окружения:
в том числе их описание, все переменные окружения и стандартные значения для некоторых из них,
можно узнать в директории `./src/core/config`

Для правильной работы приложения необходимо настроить интеграционные веб-хуки в сервисе `Jira`.
Настройку можно найти в разделе `Автоматизация`.

## Используемые технологии

* Код написан на версии **Python 3.13**
* Код приложения пишется на **Python + FastAPI**.
* Приложение запускается под управлением сервера **ASGI** (uvicorn).

### Настройка виртуального окружения

Необходима установка `Poetry`:
[Installation](https://python-poetry.org/docs/#installation)

В директории с проектом запустить команду:

```bash
poetry install
```

## Запуск тестов

Для запуская тестов с покрытием в процентах использовать

```bash
poetry run pytest --cov .
```

Для генерации HTML страниц с отчётом о покрытии использовать

```bash
poetry run pytest --cov-report=html --cov .
```

Это позволит выявить слабые места на будущее и обеспечить бэклог работой.

## Структура проекта

* Корень проекта — конфиги настройки (`.gitignore`, `.gitlab-ci.yml`, `mypy.ini` etc)
* `main.py` — входная точка приложения.
* `src` — содержит исходный код приложения.
* `api` — модуль, в котором реализуется API. Другими словами - это модуль для предоставления http-интерфейса клиентским приложениям. Разделён на версионность.
* `client` — клиенты для отправки `HTTP` запросов во внешние сервисы.
* `core` — содержит разные конфигурационные файлы.
* `schemas` - содержит классы, описывающие бизнес-сущности.
* `utils` — различные небольшие дополнения (типы, функции, маппинги и т.д.) для работы программы.
